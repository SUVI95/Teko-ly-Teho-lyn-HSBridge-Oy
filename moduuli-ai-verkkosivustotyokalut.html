Tärkein taito

Et opi työkalua. Opit ajattelemaan kuin rakentaja.

Perusymmärrys

Mitä voit rakentaa?

On hyvä ymmärtää mitä eroa on eri asioilla joita voit rakentaa. Käydään läpi kolme vaihtoehtoa konkreettisesti.

Verkkosivusto

Näyttää tietoa. Kävijä lukee mutta ei tee mitään. Ei tallenna dataa, ei reagoi käyttäjään. Esimerkki: kampaajan kotisivu jossa on yhteystiedot ja hinnasto. Hyvä esittelemään itseäsi tai yritystäsi.

Web-sovellus

Käyttäjä voi tehdä jotain. Syöttää tietoa, saada tuloksia, tallentaa asioita. Sovellus reagoi sinuun. Esimerkki: työkalu joka analysoi CV:si ja antaa parannusehdotuksia. Gmail on web-sovellus. Google Docs on web-sovellus.

SaaS

SaaS tarkoittaa Software as a Service. Se on web-sovellus josta maksetaan kuukausittain. Spotify on SaaS. Notion on SaaS. Canva on SaaS. Teknisesti se on sama asia kuin web-sovellus — ero on siinä että se on niin hyödyllinen että ihmiset maksavat siitä.

Työkalut tässä moduulissa

🔹 Lovable

Rakentaa React + Supabase -sovelluksia tekstistä. Hyvä kun haluat hallita rakennetta tarkemmin.

Hinta: Ilmainen (5 kredittiä/päivä, 30/kk) — riittää kokeiluun. Pro ~25€/kk (100 kredittiä, oma domain). → lovable.dev

🔹 Base44

Full-stack sovelluksia tekstistä. Nopea, sisältää tietokannan ja hostingin.

Hinta: Ilmainen suunnitelma (ydinominaisuudet) — riittää harjoitteluun. Maksetut alkaen ~20€/kk. → base44.com

Osa 1 – OPETAAN AI-AJATTELU (15 min)

Mistä lähteä liikkeelle?

Luovuus toimii parhaiten kun sillä on rakenne tukena. Siksi käytämme kehikkoa — se ei rajoita ideoitasi, se vapauttaa ne.

🔥 LÄMMITTELY — ennen työkaluja

Kirjoita yksi asia arjessasi joka vie aikaa tai aiheuttaa turhautumista. Voi olla pieni tai iso — esim. "Unohdan tehtävälistani", "En löydä sopivaa treeniohjelmaa", "Budjettini hajoaa joka kuukausi". Tästä syntyy sovellusideasi. Kun idea tulee omasta elämästä, sitä on helpompi rakentaa.

🧩 IDEA-KEHIKKO (täytä ennen Base44:ää tai Lovablea)

  • Mitä ongelmaa haluan ratkaista?
  • Kenelle?
  • Mitä 3 asiaa sovelluksen pitää tehdä?
  • Pitääkö siinä olla kirjautuminen?
  • Onko mukana AI-ominaisuus?

Jaa ideasi

Kerro ideastasi toiselle — tai kirjoita se ylös — omilla sanoin. Kun selität ongelmaa selkeästi, se auttaa myös promptin kirjoittamisessa. Ei tarvitse olla täydellinen: ajatuksen muotoilu ääneen on jo osa suunnittelua.

Osa 2 – Käytä ChatGPT:tä / Claudea oikein

PROMPT-POHJA (kopioi ChatGPT:hen)

Kirjoita prompt ensin, sitten kirjoita se uudestaan — vähintään 2–3 kertaa. Vertaa versioita: mitä muuttuu kun prompt paranee? Tässä oppii todella.

🔄 PROMPT-ITEROINTI

Kun AI on antanut ensimmäisen vastauksen, älä pysähdy. Kirjoita prompt uudestaan — tarkenna, lisää yksityiskohtia, kokeile eri kulmaa. Vertaa versioita vierekkäin: mitä muuttui? Miten tulos paranee? Tämä kierros on se missä oppiminen tapahtuu.

Act as a senior product architect, startup founder, and full-stack CTO. I want to build a serious, investor-ready SaaS product. You must: - Think strategically - Decide the best technical stack yourself - Add missing technical components - Suggest monetization model - Suggest scalability considerations - Structure the answer clearly - Be detailed and professional Product idea: [Describe idea in 3–5 sentences] Target market: [Who it is for] Problem it solves: [Problem] Output must include: 1. Clear product vision 2. Core features (well structured) 3. AI features (if relevant) 4. Suggested tech stack (frontend, backend, database, AI, hosting) 5. Database structure suggestion 6. Monetization model 7. User roles 8. Design direction 9. Future scalability ideas 10. Positioning statement Make it feel like a funded Nordic startup.

Miksi tämä on tärkeää

Tällainen prompt pakottaa AI:n: täyttämään puuttuvat kohdat, valitsemaan teknologisen pinon, miettimään arkkitehtuuria, miettimään rahaa (monetization) ja skaalautuvuutta. Et saa pelkkää luonnosta — saat kokonaisvaltaisen suunnitelman.

Osa 3 – Valitse rakennuspolku

Valitse yksi näistä neljästä

🔹 HARJOITUS 1 — AI-työkalu omaan arkeen

Rakenna sovellus joka auttaa sinua. Esimerkkejä: Opintosuunnittelija, Treeniohjelmageneraattori, Budjettilaskuri, CV-analysoija, Työhakemuskirjeen generaattori.

🔹 HARJOITUS 2 — Alan mukainen työkalu

Rakennat sovelluksen omaan tulevaan ammattiisi. Esimerkkejä: Rakennusalan kustannuslaskuri, Kampaamon varausjärjestelmä, Lakimiehen dokumenttitiivistäjä, HR-haastattelutyökalu, Kielten oppimistyökalu.

🔹 HARJOITUS 3 — Ratkaise suomalainen ongelma

Valitse yksi: Nuorisotyöttömyys, Maahanmuuttajien työllistyminen, Pienyritysten digitalisaatio, Mielenterveyden tukityökalu, Ajanvaraus julkisiin palveluihin.

🔹 HARJOITUS 4 — Mini-SaaS

Rakennat yksinkertaisen SaaS-idean. Pakolliset osat: Dashboard, Tietokanta, Käyttäjän syöte, Tulokset, Selkeä UI.

Osa 4 – Rakenna Base44:llä

Esimerkkiprompt Base44:ään

Build a web app called "SkillMatch AI". Purpose: Help users evaluate their skills and suggest career paths. Features: 1. User inputs their skills 2. AI analyzes strengths 3. Suggests 3 possible career paths 4. Saves previous analyses Design: Clean, minimal, professional. Light and dark mode. Mobile responsive. No complex authentication needed. Everything should save automatically.

🛠 TEHTÄVÄ

1. Näytä sovelluksesi — opettajalle tai ryhmälle. Esittele mitä rakensit ja miksi.

2. Tee vähintään 2 muutosta jotka parantavat sovellusta: esim. muuta designia, lisää uusi ominaisuus, paranna AI-prompttia sovelluksen sisällä. Näytä sovellus uudestaan muutosten jälkeen.

Osa 5 – Rakenna Lovablella

Esimerkkiprompt Lovableen

Lovable toimii paremmin kun prompt on teknisempi.

Build a SaaS web application called "Student Planner Pro". Tech: React frontend, Supabase backend, Authentication with email login, PostgreSQL database Core Features: 1. Weekly task planner 2. Task priority system 3. Completion tracking 4. Dashboard summary 5. AI assistant that suggests time management tips Design: Modern SaaS style, Minimal, Blue and white color palette, Mobile responsive Include proper database schema.

Sama tehtävä kuin Base44:llä — näytä sovelluksesi ja tee vähintään 2 parannusta.

Osa 6 – Iterointi

AI ei tee valmista tuotetta

Se tekee ensimmäisen version. Kysy AI:lta:

  • How can I improve user experience?
  • What features are missing?
  • How could this become a paid SaaS?
  • What is wrong with this UX?
Osa 7 – 3 minuutin esitys

Esitä sovelluksesi

Jokainen opiskelija esittää:

  • Mikä ongelma?
  • Kenelle?
  • Mitä sovellus tekee?
  • Mitä AI auttoi rakentamaan?
  • Mitä parantaisit?

🧠 MODUULIN YDINOPPI

AI ei korvaa ajattelua. AI: nopeuttaa, rakentaa rungon, antaa ehdotuksia. Sinä: päätät suunnan, tarkennat, parannat.

Julkaisu

Miten sovellus julkaistaan?

Kun rakennat sovelluksen Lovablella tai Base44:llä, se on heti käytettävissä — saat automaattisesti linkin jonka voit jakaa. Tämä riittää harjoitteluun ja testaamiseen.

Mutta jos haluat oikean verkkosivuston oikealla nimellä — esimerkiksi www.sinunyritys.fi — tarvitset kaksi asiaa.

1. Domain eli verkkotunnus

Domain on se osoite jonka kirjoitat selaimen yläpalkkiin. Esimerkki: google.com, yle.fi, sinunyritys.fi. Domainin voi ostaa esimerkiksi Namecheap, GoDaddy tai suomalaiselta Zoner-palvelusta. Se maksaa yleensä noin 10–15 euroa vuodessa.

2. Yhdistäminen

Kun sinulla on domain, käyt Lovablen tai Base44:n asetuksissa ja liität domainin sovellukseesi. Molemmat työkalut opastavat tämän askel askeleelta — se ei vaadi koodaamista.
